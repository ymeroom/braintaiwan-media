## 1. 目的與定位

寫三篇大眾衛教文，把「膽固醇」與「血壓」放進**神經科醫師的視角**：這兩個數字的終點不是心臟，是腦，而且腦的損害是沉默累積的（腦小血管病變、無症狀腦梗塞、血管型失智），等到出現症狀往往已不可逆。

這個角度是本系列相對於心臟科／家醫科同類衛教文的差異點，也是掛在 BrainTaiwan 品牌下的正當理由。

**明確不做的事：**

- 不寫 MD 站版本。用戶判斷這兩個題目與神經科主軸差異過大，不值得做專業導讀。
- 不開新的 MD 站分類。
- 不在本系列處理 FB 圖卡與貼文（依 2026-07-02 決定，media 站 FB 事務歸 Codex）。

## 2. 目標讀者

**主要**：健檢報告出現紅字、但還沒開始用藥的人。典型情境是「血壓 132/85，醫師說再觀察」或「總膽固醇正常、LDL 紅字，我又沒有心臟病」。

**次要**：第三篇轉向已拿到藥但因害怕副作用而沒有開始服用的人。

## 4. 三篇的內容架構

### 第一篇・血壓

- **slug**：`bp-130-brain`
- **讀者卡點**：健檢 132/85，被告知「再觀察」，於是觀察了五年。
- **核心翻轉**：台灣 2022 指引已將高血壓定義下修至 130/80 mmHg，且判準是**居家 722 量測**（每日 2 次、每次 2 讀、連續 7 天），不是診間單次數值。許多自認「沒有高血壓」的人只是量錯地方。
- **腦的終點**：收縮壓與腦小血管病變、無症狀腦梗塞、血管型失智的關係。
- **收尾**：連結既有頑固型高血壓文——「不處理的話，另一端長這樣」。

### 第二篇・膽固醇

- **slug**：`ldl-brain-target`
- **讀者卡點**：總膽固醇打勾、LDL 紅字；或「我又沒有心臟病，吃什麼藥」。
- **核心翻轉**：LDL 沒有全民通用的標準值，**目標值由風險分層決定**。同一個 LDL 數值在不同風險族群身上，結論可以完全相反。台灣 2025 血脂管理臨床路徑共識講的正是這件事。
- **腦的終點**：頸動脈與顱內動脈粥狀硬化 → 梗塞性中風。
- **必須保留的灰區**：LDL 與**出血性**中風的關係不像缺血性中風那樣單向。不迴避此點，這是本篇可信度的來源。（用戶已確認保留）

### 第三篇・怕副作用而不吃藥

- **slug**：`statin-bp-drug-fear`
- **素材來源**：`D:\claudecode\media-article-backlog\medication-fear-side-effects.md`（用戶 2026-06-23 提供的兩篇草稿）。
- **範圍收窄**：只談降壓藥與 statin 兩類，不再泛談所有藥物——因為它現在是前兩篇的收尾，不是獨立文。（用戶已確認收窄；backlog 原檔保留，日後仍可另寫其他科別版本）
- **主軸**：statin 肌肉痠痛的反安慰劑（nocebo）效應，以及降壓藥頭暈多半是劑量與服藥時間的問題。
- **落點**：不是「你該乖乖吃藥」，而是「**開始的方式可以談**」，沿用原草稿中該問醫師的那幾個問題。此落點自然導向掛號。

### 三篇的關係

第一、二篇各自獨立回答「這個數字要不要緊」，可被獨立搜尋到；第三篇回答「那我為什麼還是沒動」，靠前兩篇導流。

| 用途 | 來源 | 取得方式 |

| 膽固醇本土數字 | 2025 台灣血脂管理臨床路徑共識 | https://pmc.ncbi.nlm.nih.gov/articles/PMC11923786/ ；中文版見內科學誌 2024;35:426-430 |

**選源文的兩個原則：**

1. 以腦為終點的指引當主幹，心臟／血脂指引只當數字來源。硬套心臟科指引會讓因果鏈與標題脫節。
2. 凡屬「台灣的數字」必須取自台灣指引，不可用美國數據冒充。國際指引的目標值也不等於台灣讀者在診間遇到的現實（健保給付門檻），這個落差本身即是內容。

## 6. 產出流程

### Step 1 — 取源文與選擇性 OCR

```
D:\claudecode\cvrf-articles\
  _aha2024-prevention.pdf
  _tw-htn2022.pdf
  _tw-lipid2025.pdf
  _source-bp.md          ← 只 OCR 血壓相關章節
  _source-lipid.md       ← 只 OCR 血脂相關章節
```

先抓 PDF、依目錄定位所需章節，再以 `D:\claudecode\tools\mistral-ocr.py` 只轉需要的頁面。預估 40–50 頁，而非全部約 200 頁。

**必須用 Mistral OCR，不可用 pdftotext。** AHA 期刊樣式的 `·` 小數點與 `≥`／`≤` 符號會被 pdftotext 靜默吞掉，且句子仍然通順——這是最危險的失敗模式，naed、edh、csdh、tth 系列均已踩過。

### Step 2 — 起草

由主 agent 直接撰寫，**不外包 codex、不用 subagent 逐篇跑**。依據 `feedback_md-series-codex-drafting`：codex 會把英文引號與亂碼寫進正文；subagent 逐篇跑在 sjd 系列實測燒掉五小時只完成 1/7 篇。

文體須沿用 `_src` 既有識別：具體場景開場＋一個刺眼的數字，然後才進機轉。遵守 `feedback_writing-style` 的三不原則（不開場白、不條列、不呼籲行動）。

### Step 3 — claim-gate

所有數字型斷言逐條釘回 `_source-*.md` 的字面。預估 50–70 條。

大眾衛教文照跑閘門，理由是**錯誤數字在衛教文裡殺傷力更大**——讀者不具備查證能力。

### Step 4 — build 與檢查

1. 新增 `build-cvrf-media.js`，比照 `build-milk-media.js` 的結構（`_src/<slug>.md` → metadata 陣列帶 `desc`／`tag`／`nav` → `posts/<slug>.html`）。
2. `node --check build-cvrf-media.js`。
3. 執行三個 enhancer：`enhance-article-seo.js`、`inject-clinic-box.js`、`inject-subscribe-box.js`。
4. 本機預覽三篇。

### Step 5 — 發佈

**不自動 push、不發 FB。** build 完成後交用戶預覽，取得明確同意才推送。
